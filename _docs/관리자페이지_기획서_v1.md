## 2. 탭 1: 글 관리 — 상세 설계

### 상단 요약 바
- 전체 N개 + 카테고리별 N개 표시
- 각 항목 클릭 → 해당 카테고리 필터 즉시 적용

### 검색
- 제목 + 태그 + description 동시 검색
- 검색어 하이라이트 (`<mark>` 태그)
- 실시간 (oninput 이벤트)

### 필터 칩 (복수 선택)
- **카테고리**: 전체 / IT 정보 / 생활 정보 / 금융 정보 / 건강 정보 / 제품 추천
- **글 유형**: 전체 / 다운로드 / 바로가기 / 고객센터 / 양식
- 복수 선택 로직: 특정 항목 선택 시 '전체' 해제. 모두 해제 시 '전체' 복원.

### 태그 클릭 필터
- 카드의 태그 클릭 → 해당 태그 포함 글만 표시
- 상단에 "필터: #태그명 ✕" 표시
- ✕ 클릭 시 태그 필터 해제

### 정렬
- 최신순 / 오래된순 토글 버튼

### 글 카드 구성
```
[번호] 제목 ⚠️(이슈 있을 때)         날짜
[카테고리] [유형] [schema_type]       업데이트일
#태그1 #태그2 #태그3
description 요약
CTA N개 · FAQ ✓/🔴 · N자            [링크 복사] [글 열기→]
```

### 품질 경고 기준
| 지표 | 기준 | 표시 |
|------|------|------|
| CTA 수 | < 2개 | 🔴 |
| FAQ 여부 | 없음 | 🔴 |
| 글자 수 | < 1,000자 | 🔴 |
| last_updated | 없음 | 🟡 |

### 액션 버튼
- **링크 복사**: `navigator.clipboard.writeText(origin + url)` + 토스트 "✅ 링크 복사 완료!"
- **글 열기**: `window.open(url, '_blank')`

## 3. 탭 3: 글 작성 — 상세 설계

### 콘텐츠 건강도 뱃지
- 총 글 수: N개 / 15개 필요 (애드센스 기준) 🟢/🔴
- 카테고리별: N개 / 2개 이상 기준 🟢/🔴

### 추천 키워드 카드 (10개 하드코딩)
각 카드:
- 키워드 제목 + 카테고리/유형 뱃지
- 명령어 박스: `"키워드" 키워드로 글 써줘`
- [📋 명령어 복사] [✅ 작성완료 / ↩ 되돌리기]
- 기존 글과 키워드 유사 시 ⚠️ 경고 (앞 2단어 매칭)

### 작성완료 상태 저장
- key: `mg_kw_done` (localStorage)
- value: JSON array of completed indices
- 완료 카드: opacity 0.38

## 4. posts.json 자동 생성

### 템플릿 위치
- 소스: `/posts.json` (Liquid 템플릿, `layout: null`)
- 출력: `/_site/posts.json` (Jekyll 빌드 시 자동)

### 포함 필드
```json
{
  "title": "글 제목",
  "slug": "post-slug",
  "date": "2026-05-04",
  "last_updated": "2026-05-04",
  "category": "it",
  "type": "download",
  "schema_type": "software_faq",
  "tags": ["태그1", "태그2"],
  "description": "SEO 요약",
  "thumbnail": "/assets/images/thumbnails/slug.png",
  "url": "/posts/slug/",
  "char_count": 1445,
  "has_faq": true,
  "cta_count": 2
}
```

### 품질 지표 계산 (Liquid)
- `char_count`: `post.content | strip_html | strip | size`
- `has_faq`: `post.content contains '자주 묻는 질문'`
- `cta_count`: `post.content | split: 'class="cta-btn' | size | minus: 1`

## 5. 자체 카운터 시스템 설계 (Supabase — 미구현)

### 데이터 수집 목표
- 페이지뷰 (글별, 일별)
- CTA 클릭 (버튼별, 제휴 파트너별)
- 유입 경로 분류
- 기기/브라우저 분포

### Supabase 테이블 구조

#### `page_views` 테이블
```sql
id            bigint  generated always as identity  primary key
url           text    not null          -- /posts/slug/
slug          text    not null          -- 글 slug
referrer      text                      -- document.referrer
utm_source    text                      -- ?utm_source= 파라미터
utm_medium    text                      -- ?utm_medium=
device_type   text                      -- mobile | tablet | desktop
browser       text                      -- chrome | safari | firefox | edge | other
session_hash  text    not null          -- 중복 방지용 세션 해시
ts            timestamptz default now()
```

#### `cta_clicks` 테이블
```sql
id            bigint  generated always as identity  primary key
url           text    not null          -- 클릭이 발생한 페이지 URL
slug          text    not null
cta_text      text                      -- 버튼 텍스트
cta_type      text                      -- download | link | phone | form
affiliate     text                      -- data-affiliate 값
session_hash  text    not null
ts            timestamptz default now()
```

### 세션 해시 생성 (핑거프린트)
```javascript
const fp = [
  navigator.userAgent,
  screen.width + 'x' + screen.height,
  Intl.DateTimeFormat().resolvedOptions().timeZone,
  navigator.language
].join('|');
const hash = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(fp));
// → 30분 TTL sessionStorage 저장
```

### 중복 방지 로직
1. **세션 내 중복**: sessionStorage에 "viewed:slug" 플래그 저장. 같은 탭에서 새로고침해도 1회만 카운트.
2. **핑거프린트 기반**: 동일 기기·브라우저에서 30분 이내 재방문은 카운트 제외.
3. **관리자 제외**: `localStorage.moaguide_admin === 'true'` → 수집 중단.

### 유입 경로 분류 규칙
```
UTM 파라미터 있음 → utm_source 값 그대로 사용 (최우선)
utm_source 없음 → referrer 분석:
  - referrer 없음          → direct
  - google.com 포함        → google
  - naver.com 포함         → naver
  - daum.net 포함          → daum
  - kakao.com/t.co/fb.com  → social
  - 그 외 외부 도메인       → referral
  - 자사 도메인             → internal
```

### 수집 코드 삽입 위치
- `_layouts/post.html` 하단 `<script>` 블록
- 관리자 제외 체크: 가장 먼저 실행
- CTA 클릭: `_includes/cta-button.html`의 버튼에 이벤트 리스너 추가

## 7. 데이터 흐름

```
[Jekyll 빌드]
  _posts/*.md → posts.json (Liquid 렌더링)
  → 품질 지표 자동 계산 (cta_count, has_faq, char_count)

[관리자 접근]
  /admin.html → fetch('/posts.json')
  → 글 목록 렌더링 (필터/검색/정렬)
  → 품질 경고 인라인 표시

[방문자 (Supabase 연동 후)]
  글 페이지 방문 → 핑거프린트 생성
  → 관리자 제외 체크 → 중복 체크
  → Supabase INSERT (page_views)

[CTA 클릭 (Supabase 연동 후)]
  버튼 클릭 → cta_type + affiliate 추출
  → Supabase INSERT (cta_clicks)

[트래픽 분석 탭 (Supabase 연동 후)]
  admin.html → Supabase SELECT
  → 일별 추이 / 인기 글 TOP 10 / 유입 경로 차트
```
