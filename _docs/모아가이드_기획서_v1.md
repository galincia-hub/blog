## 1. 프로젝트 개요

### 정체성
- **이름**: 모아가이드 (MoaGuide)
- **한 줄 정의**: 구글 검색 유입 기반 수익형 정보 블로그
- **톤**: 중립적 정보 전달. 브랜드 페르소나 없음. 정보의 정확성과 구조가 핵심.
- **콘텐츠 성격**: 행동 유발형. 독자는 특정 행동(다운로드, 바로가기, 전화, 양식)을 완수하려고 검색한 사람. 글은 그 행동을 안내하는 역할.
- **레퍼런스**: IT수집가 (itsujibga.com) — 디자인, 글 구조, 카테고리 구성 참조

### 수익 구조
| 수익원 | 방식 | 목표 |
|--------|------|------|
| 애드센스 | 검색 유입 → 광고 노출 → 클릭 | 월 60만 원 |
| 제휴마케팅 | CPA/CPS 링크 삽입 | 월 40만 원 |
| **합계** | | **월 100만 원** |

- **트래픽 목표**: 일 ~1,000 PV
- **필요 글 수**: 50~80개 (안정 유입 기준)
- **현재**: 광고 요소 없이 순수 콘텐츠·구조에만 집중 (애드센스 승인 전)

## 5. 글 작성 워크플로우

키워드를 받으면 Claude Code가 자동으로:

1. **글 생성** (`_posts/YYYY-MM-DD-slug.md`)
   - writing.md 스킬 참고
   - frontmatter: title, date, last_updated, category, type, schema_type, tags, description, thumbnail
2. **썸네일 생성** (`python scripts/generate_thumbnail.py`)
   - 카테고리별 색상 자동 적용 → `assets/images/thumbnails/slug.png`
3. **CTA 버튼** 상단(도입부 직후) + 하단(마무리 직전) 각 1개씩
4. **품질 검증** (adsense.md 체크리스트 11항목)
5. **git commit & push**

### 글 구조
```
도입 2~3문장 (GEO: 검색 의도에 직접 답하는 완전한 문장)
[상단 CTA]
## 소제목1 — 표 또는 리스트
## 소제목2 — 설명 + 표
## 소제목3 (필요 시)
## 자주 묻는 질문 (FAQ)
[하단 CTA]
마무리 1~2문장
```

### frontmatter 필수 항목
```yaml
title: "글 제목"
date: YYYY-MM-DD
last_updated: YYYY-MM-DD
category: it | life | finance | health | review
type: download | link | phone | form | guide | review
schema_type: software_faq | howto_faq | article_faq
thumbnail: /assets/images/thumbnails/slug.png
description: "120자 이내 SEO 요약"
tags: [키워드1, 키워드2]
```

## 7. SEO/AEO 전략

### SEO
- Jekyll 플러그인: jekyll-sitemap, jekyll-seo-tag, jekyll-feed
- URL: 영문 슬러그 (`/posts/keyword-slug/`)
- 시멘틱 HTML: `<nav>` → `<main>` → `<article>` 구조 준수
- 내부 링크: 카테고리 페이지 + 관련 글 최대 4개 자동 연결
- 이미지 ALT: "서비스명 기능설명 스크린샷 연도" 형식

### AEO (AI 인용 최적화)
- GEO 원칙: 첫 문단이 검색 의도에 대한 완전한 답변
- FAQPage 스키마: AI 인용률 최대 67% 향상
- 비교 표: 모든 글에 최소 1개 (AI가 표 데이터 잘 파싱)
- 구체적 수치: "5가지" "3단계" 등 명시
- 날짜 명시: "2026년 5월 기준"

### Schema.org 다중 스키마
| schema_type | 삽입 스키마 |
|------------|-----------|
| software_faq | Article + FAQPage + SoftwareApplication |
| howto_faq | Article + FAQPage + HowTo |
| article_faq | Article + FAQPage |
